### 1. Latar Belakang (Alasan Membuat Aplikasi)

Dalam era digital saat ini, manajemen keuangan pribadi menjadi semakin penting untuk menjaga stabilitas finansial individu atau keluarga kecil. Banyak orang mengalami kesulitan dalam melacak pemasukan dan pengeluaran harian, yang sering kali menyebabkan ketidakseimbangan anggaran, hutang yang menumpuk, atau ketidakmampuan untuk merencanakan masa depan secara efektif. Aplikasi manajemen keuangan sederhana ini dibuat untuk mengatasi masalah tersebut dengan menyediakan alat yang mudah digunakan, berbasis desktop, dan tidak memerlukan koneksi internet.

Alasan utama pembuatan aplikasi ini meliputi:
- **Kemudahan Akses dan Penggunaan**: Banyak aplikasi keuangan modern berbasis cloud atau mobile yang memerlukan registrasi dan biaya langganan. Aplikasi ini dirancang sebagai solusi offline sederhana menggunakan Java Swing, sehingga dapat dijalankan di berbagai perangkat tanpa ketergantungan eksternal.
- **Penerapan Konsep OOP**: Sebagai proyek pendidikan atau pengembangan keterampilan, aplikasi ini secara eksplisit menerapkan prinsip-prinsip Object-Oriented Programming (OOP) seperti Inheritance (untuk hierarki transaksi), Encapsulation (untuk melindungi data sensitif seperti saldo), Polymorphism (untuk penanganan transaksi yang berbeda), dan Abstraction (untuk menyembunyikan detail implementasi kompleks).
- **Fitur Esensial untuk Pengguna Awal**: Fokus pada fitur dasar seperti pencatatan transaksi (masuk/keluar), tampilan saldo saat ini, ringkasan bulanan, dan visualisasi pie chart untuk analisis sederhana. Ini membantu pengguna pemula memahami pola pengeluaran mereka tanpa overwhelms dengan fitur lanjutan.
- **Keamanan dan Privasi**: Data disimpan secara lokal (misalnya dalam file teks atau database sederhana), sehingga pengguna tidak perlu khawatir tentang kebocoran data ke pihak ketiga.
- **Potensi Pengembangan**: Aplikasi ini dapat dikembangkan lebih lanjut, seperti integrasi dengan database eksternal atau ekspor laporan, membuatnya ideal sebagai basis untuk proyek mahasiswa atau developer pemula.

Dengan demikian, aplikasi ini tidak hanya berfungsi sebagai alat praktis untuk manajemen keuangan, tetapi juga sebagai contoh implementasi OOP yang profesional dalam bahasa Java.

### 2. UML Menggunakan Mermaid

Berikut adalah diagram kelas UML yang dirancang untuk aplikasi ini, menggunakan sintaks Mermaid. Diagram ini menunjukkan struktur kelas utama dengan penerapan OOP:

- **Abstraction**: Kelas abstrak `Transaction` menyembunyikan detail implementasi dan mendefinisikan method abstrak seperti `process()`.
- **Inheritance**: Kelas `Income` dan `Expense` mewarisi dari `Transaction`.
- **Polymorphism**: Method `process()` di-override di subclass untuk perilaku berbeda (misalnya, menambah atau mengurangi saldo).
- **Encapsulation**: Semua field bersifat private dengan getter/setter untuk akses terkendali.

Anda dapat copy-paste kode Mermaid ini ke tool seperti Mermaid Live Editor untuk visualisasi.

```mermaid
classDiagram
    class Transaction {
        -double amount
        -String date
        -String description
        +Transaction(double amount, String date, String description)
        +getAmount() double
        +setAmount(double amount) void
        +getDate() String
        +setDate(String date) void
        +getDescription() String
        +setDescription(String description) void
        +abstract process(double currentBalance) double
    }

    class Income {
        +Income(double amount, String date, String description)
        +process(double currentBalance) double
    }

    class Expense {
        +Expense(double amount, String date, String description)
        +process(double currentBalance) double
    }

    class FinanceManager {
        -double currentBalance
        -List~Transaction~ transactions
        -double monthlyIncome
        -double monthlyExpense
        +FinanceManager()
        +addTransaction(Transaction transaction) void
        +deleteTransaction(int index) void
        +editTransaction(int index, Transaction newTransaction) void
        +calculateMonthlySummary() void
        +getCurrentBalance() double
        +getTransactions() List~Transaction~
        +getMonthlyIncome() double
        +getMonthlyExpense() double
    }

    class MainApp {
        +main(String[] args) void
    }

    class DashboardPanel {
        +DashboardPanel(FinanceManager manager)
        +updateDashboard() void
    }

    class TransactionTablePanel {
        +TransactionTablePanel(FinanceManager manager)
        +refreshTable() void
    }

    class PieChartPanel {
        +PieChartPanel(FinanceManager manager)
        +updateChart() void
    }

    Transaction <|-- Income
    Transaction <|-- Expense
    FinanceManager o-- Transaction : manages
    MainApp --> FinanceManager : uses
    DashboardPanel --> FinanceManager : observes
    TransactionTablePanel --> FinanceManager : observes
    PieChartPanel --> FinanceManager : observes
```

Penjelasan singkat:
- `FinanceManager` bertindak sebagai controller utama yang mengelola data (encapsulation).
- Panel-panel GUI (view) seperti `DashboardPanel`, `TransactionTablePanel`, dan `PieChartPanel` menggunakan polymorphism untuk update berdasarkan data dari manager.
- Ini mengikuti pola MVC (Model-View-Controller) sederhana untuk pemisahan tanggung jawab.

### 3. Tampilan Sederhana yang Diperlukan (Menggunakan Swing)

Aplikasi ini akan menggunakan Java Swing untuk GUI sederhana, berbasis window utama dengan layout BorderLayout atau GridBagLayout untuk fleksibilitas. Berdasarkan gambar yang Anda berikan (dashboard dengan saldo, uang masuk/keluar, table transaksi, pie chart, dan tombol ADD/DELETE/EDIT/REFRESH/LAPORAN), berikut desain tampilan esensial:

#### Window Utama (Main Frame)
- **Title**: "Manajemen Keuangan Sederhana"
- **Size**: 800x600 piksel (resizable).
- **Layout**: BorderLayout.
  - **North**: Panel Dashboard (4 kotak: Uang Sekarang, Uang Masuk, Uang Keluar, Uang Bulan Ini).
  - **Center**: Panel Table Transaksi (JTable untuk daftar transaksi dengan kolom: Jumlah, Tanggal, Keterangan).
  - **East**: Panel Pie Chart (JFreeChart atau custom JPanel untuk pie chart menampilkan persentase Uang Masuk, Keluar, Sekarang).
  - **South**: Panel Tombol (ADD, DELETE, EDIT, REFRESH, LAPORAN).

#### Detail Komponen:
- **Dashboard Panel** (JPanel dengan GridLayout 1x4):
  - Kotak biru: Label "UANG SEKARANG" dengan JTextField (read-only) menampilkan saldo saat ini (e.g., Rp 1.348.000).
  - Kotak hijau: Label "UANG MASUK" dengan JTextField (Rp 401.000), ikon panah ke bawah.
  - Kotak merah: Label "UANG KELUAR" dengan JTextField (Rp 253.000), ikon panah ke atas.
  - Kotak oranye: Label "UANG BULAN INI" dengan JTextField (+ Rp 148.000), ikon siklus.

- **Transaction Table Panel** (JPanel dengan JScrollPane):
  - JTable dengan kolom: Jumlah (warna hijau untuk positif, merah untuk negatif), Tanggal (format YYYY-MM-DD HH:mm:ss), Keterangan.
  - Data diambil dari list transaksi di FinanceManager.

- **Pie Chart Panel** (JPanel custom menggunakan JFreeChart library):
  - Pie chart dengan slice: Uang Masuk (hijau, e.g., 20.0%), Uang Keluar (merah, 12.6%), Uang Sekarang (biru, 67.3%).
  - Legend di samping.

- **Button Panel** (JPanel dengan FlowLayout):
  - JButton: ADD (buka dialog untuk input transaksi baru: tipe (Income/Expense), amount, date, description).
  - JButton: DELETE (hapus baris terpilih di table).
  - JButton: EDIT (edit baris terpilih via dialog serupa ADD).
  - JButton: REFRESH (update semua panel).
  - JButton: LAPORAN (generate laporan sederhana, e.g., export ke teks atau tampil dialog).

#### Dialog Tambahan:
- **Add/Edit Dialog**: JDialog dengan form: JComboBox untuk tipe (Masuk/Keluar), JTextField untuk jumlah, JDateChooser untuk tanggal, JTextArea untuk keterangan, tombol Save/Cancel.

Catatan: Gunakan JFreeChart untuk pie chart (add dependency jika pakai Maven). Pastikan update real-time saat transaksi berubah (gunakan observer pattern sederhana).

### File Pendukung (yang harus Anda buat)

1. **build.bat** (Windows)
```bat
@echo off
echo Compiling...
javac -cp ".;lib/*" -d bin src/com/financeapp/*.java src/com/financeapp/*/*.java
echo.
echo Creating JAR...
jar cfe FinancialApp.jar com.financeapp.MainApp -C bin .
echo.
echo Selesai! Jalankan dengan: run.bat
pause
```

2. **run.bat** (Windows)
```bat
java -cp ".;lib/*;FinancialApp.jar" com.financeapp.MainApp
pause
```

### Cara Pakai (Super Gampang)

1. Download JFreeChart + JCommon  
   Link cepat (versi terakhir 2025):  
   → https://sourceforge.net/projects/jfreechart/files/latest/download

   Extract → ambil 2 file .jar → taruh di folder `lib/`

2. Copy seluruh struktur di atas ke komputer Anda
3. Buka folder `FinancialApp` dengan:
   - IntelliJ → Open → pilih folder ini
   - NetBeans → Open Project
   - Eclipse → Import → Existing Projects
   - VS Code + Java Extension Pack → Open Folder

4. Langsung tekan Run MainApp.java → aplikasi jalan!
